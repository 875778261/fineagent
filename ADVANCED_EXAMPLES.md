## 示例 11: 多 Agent 协作系统

**文件**: `src/example_11_multi_agent.py`  
**难度**: ⭐⭐⭐⭐⭐  
**核心概念**: Multi-Agent Collaboration

### 功能说明

模拟一个专业的旅行规划团队，包含多个专业 Agent：
- **路线规划师**: 设计最优旅行路线
- **预算分析师**: 计算和优化预算
- **当地向导**: 提供本地化建议
- **风险管理专家**: 评估安全风险
- **协调员**: 整合各方意见

### 技术亮点

1. **角色分工**: 每个 Agent 有明确的专业领域
2. **顺序协作**: Agent 按流程依次工作
3. **信息传递**: 后续 Agent 可以参考前面的结果
4. **结果整合**: 协调员综合所有意见

### 运行示例

```bash
uv run python src/example_11_multi_agent.py
```

### 应用场景

- 复杂决策系统
- 专家咨询平台
- 多角色协作任务
- 企业工作流自动化

## 示例 12: 图状工作流系统

**难度**: ⭐⭐⭐⭐⭐  
**核心概念**: Graph-based Workflow

4. **动态路由**: 可以根据结果跳转不同节点

### 运行示例

```bash
uv run python src/example_12_graph_workflow.py
```

### 应用场景

- 审批流程
- 质量检查系统
- 动态决策树

## 🔧 技术要点

### 多 Agent 系统设计

```python
class Agent:
    def __init__(self, role, expertise):
        self.role = role
        self.expertise = expertise
    
    def process(self, task, context=""):
        # 处理任务
        pass

# 创建 Agent 团队
agents = [
    Agent("专家1", "领域1"),
    Agent("专家2", "领域2"),
]

# 顺序执行
for agent in agents:
    result = agent.process(task)
```

### 工作流状态管理

```python
from typing import TypedDict

class State(TypedDict):
    current_step: str
    data: dict
    warnings: list

def node1(state: State) -> State:
    # 处理并更新状态
    state["current_step"] = "node1_done"
    return state
```

### RAG 检索优化

```python
def hybrid_search(query, docs):
    results = []
    for doc in docs:
        score = 0.0
        # 关键词匹配
        score += keyword_match(query, doc)
        # 元数据过滤
        score *= metadata_weight(doc)
        # 可靠性加权
        score *= doc.reliability
        results.append((doc, score))
    return sorted(results, key=lambda x: x[1])
```

## 💡 最佳实践

### 1. 多 Agent 协作

- **明确分工**: 每个 Agent 职责清晰
- **信息传递**: 设计好 Agent 间的数据流
- **错误处理**: 单个 Agent 失败不影响整体
- **性能优化**: 考虑并行执行

### 2. 工作流设计

- **状态完整**: 包含所有必要信息
- **节点独立**: 每个节点可独立测试
- **条件清晰**: 分支逻辑明确
- **可视化**: 绘制流程图辅助理解

### 3. RAG 系统

- **知识分层**: 不同类型知识分开管理
- **元数据丰富**: 添加足够的元信息

- **迭代控制**: 设置最大迭代次数

- **交互设计**: 友好的用户界面
